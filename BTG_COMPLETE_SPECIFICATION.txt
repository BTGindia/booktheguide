1. PLATFORM OVERVIEW
================================================================================

BookTheGuide (BTG) is a marketplace that connects travelers with local guides
across India. Guides create tour packages (products) for specific destinations,
schedule fixed departures, and accept personal booking requests. The platform
earns commission on each booking. State-level admins manage guides and approve
content for their respective states.

Key flows:
- Guides register → set up profile → create packages → schedule departures
- Admins review & approve packages and departures
- Customers search, browse, and book trips (fixed departure or personal request)
- Platform collects commission + GST on every booking

5. GEOGRAPHY SYSTEM
================================================================================

Hierarchy: State → City → Destination

States have:
- name, 2-letter code, isNorthIndia flag (for search filter grouping)
- commissionPercent (configurable per-state, default 15%)
- isActive flag

Cities belong to one state. Destinations belong to one city.

Destinations have:
- name, description, altitude (meters)
- bestMonths (recommended), openMonths (accessible), avoidMonths (dangerous/closed)
- coverImage, images array
- isActive flag

Regions are informational groupings within states (e.g., "Garhwal" in Uttarakhand).

Guides operate in ONE state with specific city selections in that state.

7. PRODUCT (TOUR PACKAGE) SYSTEM
================================================================================

--- CREATION (Guide) ---
A guide creates a product (tour package) with:

Required:
- title (min 5 chars)
- description (min 100 chars, rich text)
- destinationId (from dropdown: state → city → destination cascade)
- durationDays (min 1)
- activityType (from ActivityType enum)
- difficultyLevel (EASY/MODERATE/DIFFICULT/EXTREME)

Optional:
- durationNights (defaults to durationDays - 1)
- highlights (array of strings)
- inclusions (array of strings, e.g., "Meals", "Tent", "Guide fee")
- exclusions (array of strings, e.g., "Personal expenses", "Insurance")
- isPetFriendly (boolean)
- minAge, maxAge (age restrictions)
- coverImage (single image)
- images (gallery, up to 8)
- cancellationPolicy: array of {hours, refundPercent}
  Example: [{hours: 72, refundPercent: 100}, {hours: 48, refundPercent: 50}, {hours: 24, refundPercent: 0}]
- itinerary: day-by-day plan, array of:
  {day: 1, title: "Day Title", description: "Details", meals: "Lunch, Dinner",
   accommodation: "Guesthouse", distance: "5 km", altitude: "3000m"}
  Number of days auto-syncs with durationDays

Slug auto-generated from title (with uniqueness via timestamp hash if duplicate).

--- LIFECYCLE ---
Created → PENDING_REVIEW → APPROVED / REJECTED / CHANGES_REQUESTED
- Any edit by guide resets status to PENDING_REVIEW
- Only APPROVED products can have departures scheduled
- Products with active bookings (CONFIRMED/PENDING) cannot be deleted

--- TRENDING ---
Super Admin can toggle isTrending flag on any approved product.
Trending products appear on the homepage and dedicated /trending page.

8. FIXED DEPARTURE SYSTEM
================================================================================

--- CREATION (Guide) ---
A guide schedules a fixed departure for an APPROVED product:

Required:
- productId (must be guide's own APPROVED product)
- startDate (must be in the future)
- totalSeats (min 1, default: 15)

Optional:
- pricePerPerson (fallback default price)
- pricingTiers: {privateRoom, doubleSharing, tripleSharing, quadSharing, dormBed}
  Each is a price in currency units (e.g., INR)
- petPricing: {personWithoutPet, personWithOnePet, perAdditionalPet}
  Only shown/used if product.isPetFriendly = true
- maxGroupSize (default: 15)
- minGroupSize (default: 1)
- genderPolicy: MIXED (default) | MALE_ONLY | FEMALE_ONLY | COUPLES_ONLY
- meetingPoint, endingPoint (text)
- meetingTime (default: "06:00 AM")
- notes (additional info)

Auto-calculated:
- endDate = startDate + product.durationDays - 1

Validations:
- No overlapping departures for the same guide (across all products)
- No duplicate startDate for the same product

--- LIFECYCLE ---
Created → PENDING_APPROVAL → APPROVED / REJECTED
- Admin sets commissionPercent on approval (auto-pulled from state's commission rate)
- Only APPROVED departures accept bookings

9. BOOKING SYSTEM (TWO FLOWS)
================================================================================

=== FLOW 1: FIXED DEPARTURE BOOKING ===

Customer (or guest) books a specific departure:

Inputs:
- fixedDepartureId
- numberOfGuests (stepper +/-)
- guestDetails: [{name, age, gender}] per guest
- specialRequests (textarea, optional)
- For guests: guestEmail, guestName, guestPhone

Process:
1. Validate departure exists, is active, and approvalStatus = APPROVED
2. Check seats: remaining = totalSeats - bookedSeats; reject if < numberOfGuests
3. Calculate pricing:
   - baseAmount = pricePerPerson × numberOfGuests
   - commissionAmount = baseAmount × (commissionPercent / 100) — default 15%
   - cgstAmount = baseAmount × (cgstPercent / 100) — default 2.5%
   - sgstAmount = baseAmount × (sgstPercent / 100) — default 2.5%
   - totalAmount = baseAmount + cgstAmount + sgstAmount
   NOTE: Commission is from guide's share, NOT added to customer price
4. Create booking with status = CONFIRMED
5. Increment fixedDeparture.bookedSeats by numberOfGuests
6. Generate bookingNumber: "BTG-{YYYY}-{RANDOM5}" (e.g., BTG-2026-X7K2A)

=== FLOW 2: PERSONAL BOOKING ===

Customer requests a custom trip with a guide:

Form shows "How it works" 4-step process:
  Step 1: Choose dates & tell guide what you need
  Step 2: Guide sends you a custom package
  Step 3: Review & confirm
  Step 4: Travel!

Inputs:
- guideId (from guide profile slug)
- startDate, endDate (with real-time availability check)
- destinationName (text field)
- meetingPoint (text field)
- numberOfGuests (stepper)
- requirements: {food: bool, travel: bool, stay: bool, other: string}
- specialRequests (textarea, optional)

Process:
1. Validate startDate and endDate provided
2. Check for conflicting bookings for the guide (overlapping dates, CONFIRMED/PENDING)
3. Create booking with:
   - status = AWAITING_QUOTE
   - baseAmount = 0, commissionAmount = 0, totalAmount = 0
   - No payment at this point
4. Guide receives notification, builds custom quote

=== GUIDE QUOTE BUILDER (for Personal Bookings) ===

Guide sees customer's requirements and builds a package:

Interface:
- Shows customer details, dates, requirements
- Line item builder with fields: label, description, amount
- Quick-add suggestions: Guide Fee, Meals, Transport, Accommodation, Equipment, Permits, Porter
- Each line item has an amount; total = sum of all line items
- Grand total = total per person × numberOfGuests

Inputs:
- packageDetails: [{label: "Guide Fee", amount: 3000, description: "Per day for 5 days"}]
- totalAmount (calculated)
- commissionPercent (uses state default, typically 15%)

Process:
1. Validates booking belongs to guide, status is AWAITING_QUOTE
2. Calculates: commissionAmount = totalAmount × (commissionPercent / 100)
3. Updates booking: status → QUOTE_SENT, sets packageDetails, baseAmount, totalAmount, commissionAmount

After quote sent, customer can accept → CONFIRMED.

=== GUEST CHECKOUT ===

Both booking flows support guest checkout:
- If user is not logged in, form shows email + name + phone fields
- System finds existing user by email or creates a new passwordless CUSTOMER account
- Booking proceeds as normal using that user's ID

10. PRICING, COMMISSION & TAX FORMULAS
================================================================================

--- FIXED DEPARTURE ---
baseAmount        = pricePerPerson × numberOfGuests
commissionAmount  = baseAmount × (commissionPercent / 100)     [default 15%]
cgstAmount        = baseAmount × (cgstPercent / 100)           [default 2.5%]
sgstAmount        = baseAmount × (sgstPercent / 100)           [default 2.5%]
totalAmount       = baseAmount + cgstAmount + sgstAmount

Customer pays:  totalAmount (base + 5% GST)
Platform earns: commissionAmount (deducted from guide's share of baseAmount)
Guide earns:    baseAmount - commissionAmount

--- PERSONAL BOOKING ---
Quote-based. Guide sets totalAmount via line items.
commissionAmount = totalAmount × (commissionPercent / 100)

--- COMMISSION ---
- Per-state, configurable by Super Admin (0–50%)
- Default 15% for all states
- Auto-applied when departure is approved by admin
- Stored on each departure and each booking

--- TAX ---
- CGST = 2.5% on baseAmount
- SGST = 2.5% on baseAmount
- Total GST = 5%
- GST breakdown visible in Super Admin analytics

- comment (text, optional)

--- ADMIN REVIEWS ---
Admins/Super Admins can create reviews for guides (isAdminReview = true).
Used for bootstrapping or moderation.
Inputs: rating (1–5), comment.

--- RATING RECALCULATION ---
Triggered on every review create or delete:
1. Fetch all reviews for the guide
2. averageRating = mean(overallRating), rounded to 1 decimal
3. totalReviews = count of reviews
4. Update guideProfile.averageRating and totalReviews

--- DISPLAY ---
Guide profile page shows: average rating, total reviews, each review with sub-ratings.
Guide dashboard shows: summary stats (avg, total, positive/negative count).

13. ADMIN & APPROVAL WORKFLOWS
================================================================================

--- PRODUCT REVIEW ---
1. Guide creates product → status: PENDING_REVIEW
2. Admin sees it in "Review Products" tab (filterable: Pending/Approved/Rejected/All)
3. Admin views full product detail (itinerary, inclusions, guide info)
4. Admin approves or rejects with optional reviewNotes
5. Approved products can have departures scheduled

--- DEPARTURE APPROVAL ---
1. Guide schedules departure → status: PENDING_APPROVAL
2. Admin sees it in "Departures" tab (filterable: Pending/Approved/Rejected)
3. Shows: product name, location, guide, dates, seats, pricing, gender policy
4. Admin approves or rejects with optional notes
5. On APPROVAL: commission auto-applied from state's commissionPercent (default 15%)
6. Approved departures are bookable by customers

--- GUIDE VERIFICATION ---
Admin or Guide Manager toggles isVerified on guide profiles.
Only verified + active guides appear on personal booking pages.

Create State Admin:
  - name, email, password (min 6 chars), phone
  - Assign exactly 1 state
  - Enforces: max 1 admin per state (error 409 if state already assigned)
  - Creates User (role: ADMIN) + AdminProfile linked to state

Create Guide Manager:
  - name, email, password, phone
  - No state assignment (cross-state access)
  - Creates User (role: GUIDE_MANAGER) + GuideManagerProfile

19. AUTHENTICATION & REGISTRATION
================================================================================

--- LOGIN ---
Credentials-based (email + password) via NextAuth.js
JWT strategy with 30-day expiry
Guest accounts (no password) cannot log in via credentials

--- REGISTRATION ---
Public endpoint. Allowed roles: CUSTOMER (default) and GUIDE only.
Any other role in request defaults to CUSTOMER.

Inputs: name (required), email (required, unique), phone (optional),
        password (required, min 6 chars), confirmPassword (must match)

Password hashed with bcrypt (12 salt rounds).

--- SESSION ---
JWT carries: id, role, email, name, image
Session callback enriches session.user with id and role

--- GUEST CHECKOUT ---
If user books without account:
1. Check if email exists → use that user
2. If not → create passwordless CUSTOMER (no password, emailVerified = null)
3. Proceed with booking using that user's ID

22. KEY BUSINESS RULES SUMMARY
================================================================================

PRICING:
- Customer pays: base + 5% GST (2.5% CGST + 2.5% SGST)
- Commission deducted from guide's share, NOT added to customer price
- Commission rates: per-state, configurable 0–50%, default 15%
- Booking number format: BTG-YYYY-XXXXX

PRODUCTS:
- Created → PENDING_REVIEW → APPROVED/REJECTED
- Any edit resets status to PENDING_REVIEW
- Cannot delete if active bookings exist
- Only APPROVED products can have departures

DEPARTURES:
- Created → PENDING_APPROVAL → APPROVED/REJECTED
- Only APPROVED departures accept bookings
- No overlapping departures for same guide
- Commission auto-applied from state rate on approval
- End date auto-calculated from product duration

BOOKINGS:
- Fixed: instant CONFIRMED, seats decremented immediately
- Personal: starts as AWAITING_QUOTE → QUOTE_SENT → CONFIRMED
- Guest checkout creates passwordless account

GUIDES:
- Single operating state
- Must be verified to appear on personal booking page
- Can't mark unavailable on dates with active bookings
- Profile slug auto-generated from name

ADMINS:
- Max 1 admin per state
- State admins see only their state's data
- Super admin sees everything

RATINGS:
- Auto-recalculated on every review create/delete
- Mean of overallRating, rounded to 1 decimal
- Sub-ratings: knowledge, communication, value-for-money, safety

FILE UPLOADS:
- Max 100KB, JPEG/PNG/WebP only
- Saved to public/uploads/{folder}/{timestamp}-{random}.{ext}

AI ASSISTANT:
- Mock implementation (keyword matching, no real AI)
- Responds to keywords: trek, beach, budget, Rajasthan, etc.
- Used on destination detail pages
